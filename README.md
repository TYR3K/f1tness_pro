# 🥗 Трекер калорий — Telegram Mini App с AI-распознаванием еды

Telegram Mini App, который определяет блюдо и его калорийность (КБЖУ) **по фотографии**
с помощью искусственного интеллекта **GPT-4o Vision** от OpenAI. Пользователь фотографирует
еду, приложение распознаёт блюдо, оценивает калории, белки, жиры и углеводы — и предлагает
добавить запись в дневник питания. Также есть личный кабинет с расчётом дневной нормы калорий
и история по дням.

Приложение работает прямо внутри Telegram (как встроенное мини-приложение) и не требует
установки на телефон.

---

## ✨ Возможности

- **📷 Определение еды по фото** — сфотографируйте или загрузите снимок блюда, AI вернёт
  название, калории и БЖУ.
- **🥗 Дневник питания (рацион)** — записи по приёмам пищи (Завтрак, Обед, Ужин, Перекус),
  итоги за день и прогресс относительно вашей дневной нормы.
- **👤 Личный кабинет** — вес, рост, возраст, пол, уровень активности; автоматический расчёт
  дневной нормы калорий по формуле Миффлина — Сан Жеора; история питания за 7/30 дней.

## 🔑 (1) Как получить ключ OpenAI API

Ключ нужен, чтобы приложение могло обращаться к нейросети GPT-4o для распознавания еды.

1. Зайдите на сайт **https://platform.openai.com/** и войдите (или зарегистрируйтесь).
2. Откройте раздел **API keys**: https://platform.openai.com/api-keys
3. Нажмите **Create new secret key**, дайте ключу любое имя и нажмите создать.
4. **Скопируйте ключ сразу** — он показывается только один раз. Выглядит он примерно так:
   `sk-...` (длинная строка).
5. Пополните баланс в разделе **Billing** (https://platform.openai.com/account/billing) —
   распознавание фото платное, но стоит копейки за один снимок. Без баланса запросы к AI
   работать не будут.

Этот ключ вы позже впишете в переменную `OPENAI_API_KEY`.

> 💡 Ключ — это пароль к вашему платному аккаунту. Никому не показывайте его и не
> загружайте в публичный GitHub. Файл `.env`, где он хранится, уже добавлен в `.gitignore`.

## 🤖 (2) Как создать Telegram-бота и подключить Mini App

Mini App «живёт» внутри Telegram-бота. Сначала создаём бота, потом привязываем к нему
веб-приложение.

### Шаг 2.1. Создать бота

1. В Telegram откройте **[@BotFather](https://t.me/BotFather)**.
2. Отправьте команду `/newbot`.
3. Введите **имя** бота (отображаемое, например `Калории AI`).
4. Введите **username** бота — он должен заканчиваться на `bot` (например `my_calorie_ai_bot`).
5. BotFather пришлёт **токен** бота — длинную строку вида `123456789:AAExxx...`.
   **Скопируйте его** — это значение переменной `BOT_TOKEN`.

### Шаг 2.2. Подключить веб-приложение (Web App URL)

Сделать это можно после того, как у приложения появится публичный HTTPS-адрес
(см. раздел про деплой на Railway — там вы получите ссылку вида
`https://ваш-проект.up.railway.app`). Затем:

**Вариант А — кнопка меню (Menu Button):**
1. Отправьте BotFather команду `/mybots` → выберите своего бота.
2. **Bot Settings** → **Menu Button** → **Edit Menu Button URL**.
3. Вставьте HTTPS-ссылку на ваше приложение.
4. Задайте подпись кнопки (например, «Открыть»). Теперь в чате с ботом появится кнопка,
   открывающая Mini App.

**Вариант Б — отдельное мини-приложение (/newapp):**
1. Отправьте BotFather команду `/newapp` → выберите своего бота.
2. Заполните название, описание, загрузите иконку/картинку.
3. Когда попросят **Web App URL**, вставьте вашу HTTPS-ссылку.
4. BotFather выдаст прямую ссылку на Mini App (`https://t.me/ваш_бот/имя_приложения`).

> ⚠️ Telegram принимает **только HTTPS**-ссылки. Адрес `http://localhost:8000` для
> подключения в BotFather не подойдёт — он годится только для локального теста в браузере.

## 💻 (3) Локальный запуск на Windows

Так можно проверить приложение прямо в браузере на своём компьютере.

> Нужен установленный **Python 3.12+** (https://www.python.org/downloads/).
> При установке поставьте галочку **«Add Python to PATH»**.
>
> 💡 Если команда `python` не распознаётся, используйте штатный лаунчер **`py`**
> (ставится вместе с Python на Windows). Тогда во всех командах ниже замените
> `python` на `py` — например, `py -m venv .venv`.

Откройте **PowerShell** в папке проекта `C:\Games\MiniApp` и выполните по шагам:

```powershell
# 1. Создать виртуальное окружение
python -m venv .venv

# 2. Активировать его
.\.venv\Scripts\Activate.ps1

# 3. Установить зависимости
pip install -r backend/requirements.txt
```

> Если PowerShell ругается на запуск скриптов при активации окружения, один раз выполните:
> `Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned`, затем повторите шаг 2.

Теперь создайте файл переменных окружения из образца:

```powershell
# 4. Скопировать .env.example в .env
Copy-Item .env.example .env
```

Откройте файл `.env` в блокноте и заполните:

```dotenv
OPENAI_API_KEY=sk-ваш-ключ-openai
BOT_TOKEN=токен-вашего-бота
DATABASE_URL=sqlite:///./app.db
ALLOW_INSECURE_AUTH=1
```

> ⚠️ **ВАЖНО для локального теста в браузере:** поставьте `ALLOW_INSECURE_AUTH=1`.
> Вне Telegram приложение не получает подпись `initData`, и без этого флага сервер
> ответит ошибкой авторизации (401). Флаг включает «тестового пользователя» для разработки.
> **На реальном сервере (Railway) обязательно верните `ALLOW_INSECURE_AUTH=0`**, иначе
> авторизация будет небезопасной.

Запустите сервер:

```powershell
# 5. Запуск с автоперезагрузкой при изменении кода
uvicorn backend.main:app --reload
```

Откройте в браузере **http://localhost:8000** — приложение загрузится.
Распознавание фото будет работать, если в `.env` указан рабочий `OPENAI_API_KEY` с балансом.

## 🚀 (4) Деплой на Railway за 5 минут

[Railway](https://railway.app) — простой хостинг, который сам соберёт и запустит проект.

1. **Залейте проект на GitHub.** Создайте репозиторий и отправьте код:
   ```powershell
   git init
   git add .
   git commit -m "Трекер калорий — Telegram Mini App"
   git branch -M main
   git remote add origin https://github.com/ВАШ_ЛОГИН/ВАШ_РЕПОЗИТОРИЙ.git
   git push -u origin main
   ```
   > Файл `.env` с секретами **не попадёт** в репозиторий — он в `.gitignore`. Это правильно.

2. **Создайте проект на Railway.** Зайдите на https://railway.app, нажмите
   **New Project** → **Deploy from GitHub repo** → выберите свой репозиторий.
   Railway увидит `railway.toml` и `requirements.txt`, поймёт, что это Python, и сам всё соберёт.

3. **Добавьте переменные окружения.** Откройте сервис → вкладка **Variables** → добавьте:
   - `OPENAI_API_KEY` — ваш ключ OpenAI;
   - `BOT_TOKEN` — токен бота из BotFather;
   - `DATABASE_URL` — `sqlite:///./app.db` (см. предупреждение ниже про хранение данных);
   - `ALLOW_INSECURE_AUTH` — `0` (на проде авторизация только через Telegram).

   Переменную `PORT` задавать **не нужно** — Railway подставит её сам.

4. **Получите домен.** Откройте **Settings** сервиса → раздел **Networking** →
   **Generate Domain**. Появится адрес вида `https://ваш-проект.up.railway.app`.

5. **Подключите этот HTTPS-адрес в BotFather** (см. раздел 2.2 — Menu Button или /newapp).

После пуша новых коммитов в GitHub Railway автоматически пересоберёт и перезапустит проект.

### ⚠️ ВАЖНОЕ ПРЕДУПРЕЖДЕНИЕ о хранении данных (SQLite на Railway)

По умолчанию база — это файл SQLite (`app.db`). **На Railway файловая система эфемерна:**
при каждом редеплое (новый коммит, перезапуск) файл `app.db` **создаётся заново, а все
записи теряются**. Для теста это нормально, но для постоянного хранения выберите один из вариантов:

- **Railway Volume (постоянный диск).** В настройках сервиса добавьте **Volume**, примонтируйте
  его, например, в `/data`, и поставьте переменную:
  `DATABASE_URL=sqlite:////data/app.db`
  (четыре слэша — это абсолютный путь `/data/app.db`). Теперь база переживёт редеплои.
- **PostgreSQL.** Подключите базу Postgres (плагин Railway Postgres или бесплатный тариф
  **Supabase**: https://supabase.com) и пропишите её строку подключения в `DATABASE_URL`
  (вид `postgresql://...`). Это самый надёжный вариант для продакшена.

## ✅ (5) Как проверить, что всё работает

1. **Проверка сервера (health-check).** Откройте в браузере:
   - локально: `http://localhost:8000/api/health`
   - на Railway: `https://ваш-проект.up.railway.app/api/health`

   Должен вернуться ответ:
   ```json
   {"status":"ok"}
   ```
   Значит, сервер запущен и отвечает.

2. **Откройте приложение в Telegram.** Зайдите в чат с вашим ботом и нажмите кнопку меню
   (или откройте ссылку Mini App от BotFather). Приложение должно загрузиться с тремя
   вкладками внизу: **📷 Определение**, **🥗 Мой рацион**, **👤 Мой аккаунт**.

3. **Сделайте фото еды.** На вкладке **«Определение»** нажмите
   «Сфотографировать / Загрузить», выберите снимок блюда. Через пару секунд появится
   карточка с названием, калориями и БЖУ. Нажмите «Добавить в рацион» — запись окажется
   в дневнике на вкладке **«Мой рацион»**.

4. **Заполните профиль.** На вкладке **«Мой аккаунт»** укажите вес, рост, возраст и пол,
   нажмите «Рассчитать автоматически» — приложение посчитает дневную норму калорий.
   Сохраните. На экране «Мой рацион» появится прогресс относительно этой нормы.

## 🗂️ Краткий справочник API

Все ответы — в формате JSON. Запросы (кроме `/api/health`) требуют заголовок
`X-Telegram-Init-Data` с подписью Telegram WebApp.

| Метод и путь            | Назначение                                              |
|-------------------------|---------------------------------------------------------|
| `GET  /api/health`      | Проверка живости сервера (без авторизации)              |
| `POST /auth/verify`     | Проверка/создание пользователя, возврат профиля         |
| `POST /food/analyze`    | Анализ фото еды (поле формы `file`)                      |
| `POST /diary/add`       | Добавить запись в дневник                                |
| `GET  /diary/{date}`    | Дневник за день (`YYYY-MM-DD`)                           |
| `DELETE /diary/{id}`    | Удалить запись дневника                                  |
| `GET  /profile`         | Получить профиль                                         |
| `POST /profile`         | Сохранить профиль (и пересчитать норму калорий)          |
| `GET  /history?days=30` | История калорий по дням                                 |

### Переменные окружения

| Переменная            | Назначение                                                                 |
|-----------------------|---------------------------------------------------------------------------|
| `OPENAI_API_KEY`      | Ключ доступа к OpenAI (распознавание еды)                                  |
| `BOT_TOKEN`           | Токен Telegram-бота (проверка подписи initData)                           |
| `DATABASE_URL`        | Строка подключения к БД (по умолчанию `sqlite:///./app.db`)               |
| `ALLOW_INSECURE_AUTH` | `1` — тестовый пользователь без Telegram (ТОЛЬКО для разработки); `0` — выкл |
| `DEBUG_AI`             | `1` — показывать «сырой» ответ GPT в приложении и в тексте ошибки (отладка распознавания); `0` — выкл |
| `OPENAI_MODEL`         | Необязательно. Модель распознавания (по умолчанию `gpt-4o`)                |

> 🔍 **Отладка распознавания.** «Сырой» ответ модели всегда пишется в логи сервера
> (на Railway — вкладка **Logs**). Чтобы видеть его прямо в приложении (блок
> «Ответ модели (debug)» на экране результата и подробности в тексте ошибки),
> поставьте `DEBUG_AI=1`. В dev-режиме (`ALLOW_INSECURE_AUTH=1`) отладка включена
> автоматически. Если еда часто «не распознаётся» — посмотрите там, что именно
> вернул GPT, и при необходимости поменяйте `OPENAI_MODEL` или промпт в
> `backend/ai_service.py`.
