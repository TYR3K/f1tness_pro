# Подключение Tribute (второй способ оплаты)

Tribute — это сервис монетизации внутри Telegram (бот **@tribute**): подписки,
цифровые продукты, донаты. Оплата проходит в Telegram, поэтому Tribute знает
Telegram-аккаунт плательщика — это удобно для активации премиума нужному человеку.

> **Основной способ оплаты в приложении — Telegram Stars** (работает «из коробки»,
> внутри Mini App). Tribute — дополнительный способ. Если не хотите его подключать —
> просто оставьте `TRIBUTE_URL` и `TRIBUTE_WEBHOOK_SECRET` пустыми, и кнопка Tribute
> не будет показываться.

---

## 1. Регистрация в Tribute

1. В Telegram откройте бота **[@tribute](https://t.me/tribute)** и нажмите **Start**.
2. Пройдите регистрацию автора/продавца (Tribute попросит базовые данные,
   возможно — привязку для вывода средств; следуйте подсказкам бота).
3. Откройте панель управления Tribute (бот даёт ссылку на веб-кабинет / меню).

## 2. Создание продукта (тарифов)

В кабинете Tribute создайте платные продукты под ваши тарифы:

| Тариф    | Что создать в Tribute                         | Цена (ориентир) |
|----------|-----------------------------------------------|-----------------|
| Месячный | Подписка с периодом 1 месяц                   | ~500 ₽          |
| Годовой  | Подписка с периодом 1 год                     | ~годовая цена   |
| Вечный   | Разовый цифровой продукт (lifetime)           | ~разовая цена   |

- Для каждого продукта Tribute выдаёт **ссылку на оплату** (вида `https://t.me/tribute/app?...`
  или `https://tribute.tg/...`). Эту ссылку положите в `.env` как `TRIBUTE_URL`
  (для начала — ссылку основного тарифа; при желании можно сделать отдельный экран
  с несколькими ссылками позже).

## 3. Webhook (уведомление о платеже)

1. В настройках Tribute найдите раздел **Webhooks / API / Интеграции**.
2. Укажите URL вашего вебхука:
   ```
   https://<ваш-домен>.up.railway.app/payment/tribute/webhook
   ```
3. Включите события об **успешной оплате / новой подписке** (`new_subscription`,
   `payment_succeeded` — точное имя зависит от Tribute).
4. Tribute покажет **секрет/ключ подписи** вебхука — скопируйте его в `.env`:
   ```
   TRIBUTE_WEBHOOK_SECRET=<секрет из Tribute>
   ```

**Как наш бэкенд проверяет и читает вебхук** (`backend/main.py` → `/payment/tribute/webhook`):
- Секрет сверяется по заголовку (`X-Tribute-Signature` / `Authorization` / `X-Webhook-Secret`)
  **или** по полю тела (`secret` / `signature`). Если `TRIBUTE_WEBHOOK_SECRET` пуст — проверка
  пропускается (только для локальной отладки).
- Telegram-id плательщика берётся гибко: из `telegram_id` / `user_id` верхнего уровня
  **или** из `metadata.telegram_id` / `data.user_id`.
- Тариф — из `tariff` / `product` (если не распознан, по умолчанию `monthly`).
- В ответ всегда отдаём `200 OK` (чтобы Tribute не ретраил), активация — внутри.

> ⚠️ Точные имена полей и механизм подписи у Tribute со временем меняются. Сверьте
> формат вебхука в текущем кабинете Tribute. Парсер у нас намеренно гибкий, но если
> поле с id называется иначе — допишите его в список в `payment_tribute_webhook`
> (одна строка) или напишите мне на Этапе доработки.

## 4. Связка платежа с пользователем (telegram_id)

Поскольку Tribute — это Telegram-бот, **он знает Telegram-аккаунт плательщика**, и в
вебхуке обычно присылает его `telegram_user_id`. Сценарий:

1. Пользователь в Mini App открывает экран **Подписка** → жмёт «Оплатить через Tribute».
2. Открывается ссылка Tribute, пользователь платит **своим** Telegram-аккаунтом.
3. Tribute шлёт вебхук с его `telegram_id` → бэкенд активирует премиум этому id.

**Честный нюанс / fallback:** если в вашей версии Tribute вебхук **не** содержит
`telegram_id` плательщика (или вы используете оплату картой вне Telegram), автоматически
сопоставить платёж с пользователем нельзя. Варианты:
- **A (рекомендуется).** Использовать Tribute именно как Telegram-подписку — тогда id
  плательщика приходит в вебхуке, и всё активируется автоматически.
- **B.** Передавать `telegram_id` через параметр/utm в ссылке оплаты (если Tribute
  возвращает его в `metadata` вебхука) — тогда формируйте `TRIBUTE_URL` с подстановкой id.
- **C (ручной).** Владелец (@TYREK223) выдаёт премиум вручную командой `/givepro <username>`
  после подтверждения оплаты. Это всегда работает как запасной путь.

## 5. Что добавить в `.env`

```
TRIBUTE_WEBHOOK_SECRET=<секрет вебхука из Tribute>
TRIBUTE_URL=<ссылка на оплату из Tribute>
```

После изменения переменных на Railway сервис перезапустится автоматически.

## 6. Как протестировать

1. Задеплойте с заполненными `TRIBUTE_*`, откройте Mini App → **Подписка** →
   должна появиться кнопка «Оплатить через Tribute».
2. Сделайте тестовую оплату (минимальный тариф) **своим** Telegram-аккаунтом.
3. Проверьте **Railway → Logs**: должен прийти POST на `/payment/tribute/webhook`
   (и запись об активации). Если секрет не совпал — в логах будет «неверный секрет».
4. В приложении статус на экране «Подписка» станет **активным**, платные разделы
   (Тренировки/Добавки) откроются.
5. Если активация не сработала — посмотрите в логах реальный payload вебхука и
   сверьте, где лежит `telegram_id`/тариф (см. п.3 про гибкий парсер).

> Если что-то из Tribute не укладывается в эту схему — напишите, и я подгоню парсер
> вебхука под фактический формат на следующем шаге.
